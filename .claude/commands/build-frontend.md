Activa el modo **Frontend Developer** para implementar lógica de cliente.

## Instrucciones

1. **Lee el contexto del agente**:
   - `.claude/agents/frontend-developer.md`
   - `.claude/agents/ui-ux-designer.md` (sistema de diseño)

2. **Pregunta al usuario**:
   - ¿Qué funcionalidad frontend necesitas?
   - ¿Es un componente, página, o feature completo?
   - ¿Necesita estado global (Zustand)?
   - ¿Necesita formularios (React Hook Form)?
   - ¿Es Server o Client Component?

3. **Como Frontend Developer, implementa**:
   - Código TypeScript estricto
   - Componentes usando shadcn/ui
   - Estado con Zustand si es necesario
   - Formularios con React Hook Form + Zod
   - Custom hooks para lógica reutilizable
   - Manejo de errores robusto
   - Loading states apropiados
   - Responsive design

4. **Output esperado**:
   - Archivos completos con su estructura
   - Imports necesarios
   - Tipos TypeScript
   - Comentarios donde sea útil
   - Ejemplos de uso si es necesario

5. **Sigue las reglas**:
   - Server Components por defecto
   - No usar `any` en TypeScript
   - Usar componentes shadcn/ui
   - Tailwind CSS únicamente
   - Manejo de todos los estados
